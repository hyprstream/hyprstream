## #321 — AEAD + provenance on mesh payloads (capnp gate)
**State:** moq mesh plane is plaintext + chained HMAC (integrity-only); `derive_stream_keys` has no
`enc_key` (`key_exchange.rs:136`); but `TaggedPayload` slot + serialize arm (`streaming.rs:850`) + decode
arm (`stream_consumer.rs:250`) ALL exist, and `event_crypto::encrypt_event` (`event_crypto.rs:254`) returns
exactly the 4 TaggedPayload fields. `derive_notification_keys` already has the `enc_key` pattern to copy
(`key_exchange.rs:213`). `StreamBlock.epoch` exists for rekey.
- **AEAD wire-up = PURE-RUST, NO schema change** (decided MANDATORY, D-4): add `enc_key` to
  `derive_stream_keys` (context `"hyprstream stream-keys v1 enc"`, Zeroizing), encrypt at
  `moq_stream.rs:write_block` → emit `Tagged`, decrypt in the `Which::Tagged` arm via `decrypt_event_full`.
  Bind nonce/key-commitment to `epoch`.
- **Provenance (C-PROV) = the ONLY capnp gate.** Stream blocks have no per-host signer field (only the
  HMAC chain, which proves "held DH key + in order", not "host-X computed this", threat T3). In-band
  provenance needs an additive `StreamBlock.provenance @N :Provenance{signerKid @0 :Data; sig @1 :Data}`,
  signed with #328's per-host hybrid COSE, verified via the mesh_peers roster.
  - (a) **defer provenance to Phase-4 pipeline.capnp [REC]** — its primary threat (rogue *stage*) is P4/2c;
    today's mesh stream carries tokens/deltas, not activations. Ship AEAD-only now.
  - (b) add the `Provenance` field now — promote to this if cross-host **delta submission** ships in Phase 2
    (a poisoned delta needs a signer to attribute, C-AGG/T4).
